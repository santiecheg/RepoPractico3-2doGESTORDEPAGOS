Ejercicio 2 — Sistema de Procesamiento de Pagos

Una empresa necesita un sistema de consola para procesar distintos métodos de pago utilizados por sus clientes.

El sistema debe permitir:
• Registrar pagos realizados con distintos medios.
• Mostrar la información de cada pago.
• Procesar operaciones de pago.
• Calcular costos adicionales o comisiones dependiendo del tipo de pago utilizado.
• Mostrar el monto final de la operación.
Requisitos funcionales:

• Cada pago debe almacenar información básica del titular y del monto.
• El sistema debe contemplar distintos métodos de pago.
• Cada método de pago debe tener una forma distinta de procesar la operación y calcular sus cargos adicionales.
• Desde el programa principal se deben crear distintos pagos y ejecutar sus operaciones.
El programa debe demostrar:
• Diseño orientado a objetos.
• Uso de herencia y polimorfismo.
• Separación de responsabilidades.
• Implementaciones específicas según el tipo de objeto.
